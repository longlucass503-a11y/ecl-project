# ECL 引擎层 — 6.5 LGD 计算引擎 设计说明

> **日期**：2026-06-21 | **依据**：技术设计说明书 Section 6.5 | **分支**：`feature/engine-layer`

## 1. 目标

实现 LGD 计算引擎（6.5），从 LGD 曲线表中按 (groupId, collateralType, productType) 查基准值。缺值时依次走 NONE 路径和方案 defaultLgd 兜底。当前简化为借据级（无押品池）。

## 2. 新增文件

```
ecl-engine/src/main/java/com/bank/ecl/engine/lgd/
└── LgdEngine.java
```
修改：`core/AssetInput.java`（新增 lgdValue, lgdException）

## 3. AssetInput 扩展

| 字段 | 类型 | 来源 |
|------|------|------|
| lgdValue | double | 6.5 输出 |
| lgdException | String | 6.5 输出（"WARN"=兜底路径） |

## 4. LgdEngine

- Mapper: `LgdCurveMapper`
- 缓存 key: `groupId + "|" + collateralType + "|" + productType`
- 查找顺序: 精确匹配 → NONE 路径 → defaultLgd（兜底）
- 兜底时标记 `lgdException = "WARN"`

## 5. 测试场景

| 场景 | 预期 |
|------|------|
| 精确匹配 | lgdValue = lgd_base_value |
| 精确缺值 + NONE 命中 | lgdValue = NONE 路径的值 |
| 全部缺值 | lgdValue = defaultLgd, exception="WARN" |
| LGD 曲线为空 | 所有借据用 defaultLgd |
